# IP 黑白名单查询工具

基于本地 CSV 的 IP 黑白名单查询脚本，支持单个 / 批量查询，精确匹配 + 模糊网段匹配，同时兼容 IPv4 与 IPv6。提供**对话模式**（推荐，免命令行参数）与**命令行模式**两种用法。

> **说明**：仓库内 `黑名单.csv` / `白名单.csv` 仅为示例数据

## 特性

- 数据源为本地 `黑名单.csv` / `白名单.csv`，脚本只读不写（除非显式 `add`）。
- **精确匹配**：单 IP 相等，或查询 IP 落在名单中某个 CIDR 网段内。
- **模糊匹配**：取查询 IP 的前 2 段 / 前 3 段超网，检查名单中是否存在同网段条目（用于「同段相关」预警）。
  - IPv4：`/16`（前 2 段）、`/24`（前 3 段）
  - IPv6：`/32`（前 2 段）、`/48`（前 3 段）
- 同时支持 IPv4 与 IPv6（含 IPv6 CIDR 网段）。
- 脏数据容错：自动清洗全角逗号、尾部空格；超大网段仅告警不崩溃。
- 冲突检测：若同一 IP 在黑白名单中同时精确命中，会标 `⚠ 冲突`（白名单优先，但请人工复核）。
- 纯 Python 标准库实现，无需安装任何依赖。

## 环境要求

- Python 3.8+（已在 Python 3.12 / 3.13 验证）
- 无需 `pip install`，仅用到标准库：`argparse` / `csv` / `ipaddress` / `os` / `select` / `sys`

## 数据源格式

脚本读取当前目录下的两个 CSV 文件：

**黑名单.csv**（表头：`序号,封禁IP`；查询只认 IP 列）
```
序号,封禁IP
1,192.0.2.10
```

**白名单.csv**（表头：`序号,白名单`）
```
序号,白名单
1,192.0.2.55
2,198.51.100.0/24
3,2001:db8:100d:ffac::/64
```

- 白名单支持：**IPv4 单 IP** / **IPv4 CIDR** / **IPv6 CIDR**。
- 手动维护这两个 CSV 即可，改完在对话模式输入 `reload` 立即生效。

## 快速开始（对话模式，推荐）

直接运行，无需任何参数：

```bash
python3 ip_query.py
```

进入对话模式后：

```
========================================================
IP 黑白名单查询 · 对话模式
========================================================
黑名单: 黑名单.csv (18 条)    白名单: 白名单.csv (12 条)
输入 IP，每行一个；空行 + 回车 执行查询（直接粘贴多行也行）
输入 help 查看命令，exit 退出
========================================================
> 192.0.2.10
198.51.100.7
8.8.8.8

[192.0.2.10]  (IPv4)
  黑名单: 精确命中 -> 192.0.2.10
  白名单: 精确命中 -> 192.0.2.0/24
  ⚠ 冲突（白名单优先，请复核）
...
--------------------------------------------------------
共 3 个 | 黑精确 2 | 黑模糊 0 | 白精确 2 | 白模糊 0 | ⚠冲突 2
```

- 每行输入一个 IP，**空行 + 回车**执行这一批查询（也可直接粘贴多行，自动整批执行）。
- 执行完可继续输入下一批，无需重启。

### 对话模式命令

| 命令 | 说明 |
| --- | --- |
| （直接输入 IP） | 每行一个，空行 + 回车执行批量查询 |
| `go` | 立即执行已输入的 IP（与空行等效） |
| `add` | 交互式录入到黑白名单（按提示输入类型与 IP） |
| `add <black\|white> <IP/CIDR ...>` | 单行录入，如 `add white 203.0.113.0/28` |
| `reload` | 重新读取 CSV（手动改完文件后用，不清空已输入内容） |
| `clear` | 清屏 |
| `help` | 显示命令一览 |
| `exit` / `quit` | 退出 |

### 查完后追加黑名单（仅对话模式）

每批查询结束后会询问：

| 选项 | 含义 |
| --- | --- |
| `a` 全量追加 | 本批全部可解析 IP 写入黑名单（允许与现有条目重复） |
| `b` 去重追加 | 跳过已在黑名单**精确命中**的，只写入其余 |
| `c` 不追加 | 不写入 |

管道 / `query` 子命令**不会**追问，只输出结果。

## 命令行模式

不进入对话模式时，也可使用子命令（与对话模式二选一，按个人习惯）：

```bash
# 查询：多 IP 空格分隔
python3 ip_query.py query 192.0.2.10 198.51.100.7 8.8.8.8

# 从文件批量查询（每行一个 IP）
python3 ip_query.py query -f ips.txt

# 从管道多行输入
cat ips.txt | python3 ip_query.py query

# 结果同时写入文件
python3 ip_query.py query -f ips.txt -o result.txt

# 可选：向黑名单 / 白名单追加录入
python3 ip_query.py add black 203.0.113.1
python3 ip_query.py add white 203.0.113.0/28
python3 ip_query.py add black -f newips.txt
```

## 匹配规则说明

| 类型 | 判定 | 示例 |
| --- | --- | --- |
| 精确命中（黑名单） | IP 等于某条目，或落在某 CIDR 内 | `192.0.2.10` 命中黑名单单 IP |
| 精确命中（白名单） | 同上，并提示「建议放行」 | `198.51.100.7` 命中白名单 `198.51.100.0/24` |
| 模糊命中（黑名单） | 同 `/16` 或 `/24` 网段存在黑名单条目 | `192.0.2.200` 命中同网段黑名单条目 |
| 模糊命中（白名单） | 同网段存在白名单条目，提示「需人工确认」 | IPv6 `2001:db8:9999::1` 命中白名单文档前缀 `/32` |
| 冲突 | 黑白同时精确命中 | 标 `⚠ 冲突`，白名单优先但需复核 |

## 容错与已知注意点

- **脏数据清洗**：`192.0.2.10，,`（全角逗号混入）、`198.51.100.7 `（尾部空格）等均可正确解析。
- **超大网段告警**：白名单中若出现 `/2`、`/1` 等超宽网段，会打印 `⚠ [警告]` 但不影响运行。
- 演示数据仅使用合理前缀。若自建名单出现 `/2`、`/1` 等超宽网段，脚本会告警；改 CSV 后对话模式输入 `reload` 即可。
